## Project Overview

This is an iTerm2 color theme inspired by [Tokyo Night](https://github.com/tokyo-night/tokyo-night-vscode-theme) and [Catppuccin](https://github.com/catppuccin/catppuccin). There is no build system, package manager, or test suite — it is a pure static asset repository.

## Repository Structure

- `tokyo-night.itermcolors` — Original theme (Apple plist/XML format).
- `tokyo-night-v2.itermcolors` — v2 theme with improved contrast and distinct normal/bright pairs.
- `tokyo-night-v2.yaml` — Human-readable export of the v2 palette. **Auxiliary only** — not a build input. The `.itermcolors` files are the canonical source of truth for all color values.
- `starship.toml` — Starship prompt config themed to the v2 palette.
- `tokyo-night-bg.png` — 4K (3840×2160) iTerm2 background for the original theme. Generated by `gen_bg.py`.
- `tokyo-night-v2-bg.png` — 4K iTerm2 background for v2. Generated by `gen_bg2.py`.
- `chrome-theme/` — Chrome browser theme extension. Generated by `gen_chrome_theme.py`.
- `colors/` — PNG swatches (16×16 px) for each of the 16 ANSI colors, used in README.md.
- `example_1.png` — Screenshot used in README.md.
- `README.md` — Color palette tables and installation instructions.
- `Makefile` — `make check` / `make regen` for verifying and regenerating PNG assets.

## Color Format

`.itermcolors` files use Apple's plist XML format. Colors are stored as floating-point sRGB components (0.0–1.0) for Red, Green, Blue, and Alpha. To convert from hex: divide each 8-bit channel by 255.

The defined color keys are:
- `Ansi 0`–`Ansi 7`: Normal ANSI colors (Black, Red, Green, Yellow, Blue, Magenta, Cyan, White)
- `Ansi 8`–`Ansi 15`: Bright variants of the above
- `Background Color`, `Foreground Color`, `Bold Color`, `Cursor Color`, `Cursor Text Color`, `Cursor Guide Color`, `Selection Color`, `Selected Text Color`, `Link Color`, `Badge Color`, `Tab Color`

## Generated Assets

PNG files are generated by Python scripts using Pillow + numpy. After editing a generator:

```sh
make regen   # regenerate in-place and stage
make check   # verify committed PNGs match generators (non-destructive, CI-safe)
```

| Asset | Generator |
|---|---|
| `tokyo-night-bg.png` | `gen_bg.py` |
| `tokyo-night-v2-bg.png` | `gen_bg2.py` |
| `chrome-theme/manifest.json` + `chrome-theme/images/*.png` | `gen_chrome_theme.py` |
